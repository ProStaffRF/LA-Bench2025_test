LA-Bench 2025: 実験手順生成AI - 提出モデル解説
📝 概要 (Overview)
本提出物は、LA-Bench 2025 コンペティションのベースライン（baseline.ipynb）に対し、**システムプロンプトの最適化（Prompt Optimization）**を行うことで、ラボオートメーション（LA）システムが実行可能な詳細な実験手順を生成できるように改良したものです。

複雑な外部知識ベースを使用せず、LLMへの指示を**「役割」「制約」「具体性」**の3点に絞って厳格化することで、汎用性と実行可能性を高めています。

💡 アプローチ (Approach)
本モデルでは、以下のシステムプロンプトを採用し、生成精度の向上を図りました。

sys = (
    "あなたは生命科学実験の専門家であり、ラボオートメーションシステム用の手順書作成者です。以下の Input を読み、"
    "日本語で実行可能な、**曖昧さのない詳細な**実験手順（procedure_steps）を返してください。"
    "制約: ステップ数は最大50、各ステップは10文以下、idは1から昇順。**あなたはJSON形式のデータ構造（procedure_steps）のみを出力し、それ以外の説明文や前置きを一切含んではいけません。**"
    "**手順には、使用する試薬の量や反応時間など、自動実行に必要な全てのパラメータを具体的に記述すること。**"
)

1. 役割の明確化 (Role Definition)
単なるAIアシスタントではなく、**「ラボオートメーションシステム用の手順書作成者」**という役割を与えることで、人間向けの曖昧な手順ではなく、機械が実行可能なロジカルな手順の生成を促しています。

2. 具体性の強制 (Enforcing Specificity)
自然言語の実験指示には「適量」「しばらく」といった曖昧さが含まれます。プロンプトで**「自動実行に必要な全てのパラメータを具体的に記述すること」**と明示することで、試薬量や反応時間などの数値化をモデルに強制し、Specific Scoreの向上を狙っています。

3. 出力形式の厳格化 (Strict Output Format)
後処理のエラーを防ぐため、**「JSON形式のデータ構造のみを出力し、それ以外の説明文や前置きを一切含んではいけません」**という制約を設け、評価システムが確実に解析できる形式で出力させます。
